# Требования на дизайн · SCR-006 · Детали брони + отмена

**ID:** SCR-006 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Foundations](00-foundations.md) · [ФТ](../functional-requirements.md) · [Use cases](../use-cases.md)

---

## 1. Назначение и контекст
Полная информация об одной брони + точка входа в отмену (UC-2) и в оценку инструктора (UC-4) для завершённых тренировок.

## 2. Навигация
### Входящие
- Карточка брони на [SCR-005](SCR-005-my-bookings.md).
### Исходящие
- «Отменить запись» → [BS-003](BS-003-cancel-confirm.md).
- «Оценить инструктора» (для завершённых) → [BS-004](BS-004-rate-instructor.md).

## 3. Контент-инвентарь
| # | Элемент | Источник | Назначение |
|---|---|---|---|
| 1 | Статус брони + причина (если отменена скалодромом) | `booking.status`, `booking.cancel_reason` | FR-29. |
| 2 | Дата/время, зона/формат, инструктор | `slot.*` | Сводка. |
| 3 | Снаряжение и тариф проката | `booking.equipment` | — |
| 4 | Правило отмены (текст) | foundations §6 | Информирование перед действием. |
| 5 | Кнопка «Отменить запись» (только для активных, не начавшихся) | — | → BS-003 |
| 6 | Блок оценки инструктора (только для завершённых) | `booking.rating` | Показывает уже поставленную оценку или CTA «Оценить» |

## 4b. Структура (завершена)
```
┌─────────────────────────────┐
│ ✓ Завершена                   │
│ Чт, 3 июля · 19:00              │
│ Трассы · Игорь                 │
│ ─────────────────────────    │
│ Ваша оценка: ★★★★☆ «Отличный  │
│ инструктор!»                   │
└─────────────────────────────┘
```
Если оценки ещё нет — вместо блока оценки показывается кнопка «Оценить инструктора» → BS-004.

## 5. Компоненты и поведение
- Кнопка «Отменить запись» видна только для статуса «активна» и если тренировка ещё не началась.
- После отмены (через BS-003) экран обновляет статус на месте (без перехода на другой экран).
- Оценка после отправки — доступна только для чтения (нельзя изменить, FR-40).

## 6. Валидации и микрокопия
| Ситуация | Текст |
|---|---|
| Правило отмены | из foundations §6, дословно |
| Поздняя отмена (после действия) | из foundations §6.1 |
| Тренировка уже началась | кнопка «Отменить запись» отсутствует |

## 7. Критерии приёмки
```gherkin
Функция: Детали брони и отмена (SCR-006)

  Сценарий: AC-001 — доступна отмена
    Дано бронь активна и тренировка не началась
    Тогда видна кнопка «Отменить запись», ведущая на BS-003

  Сценарий: AC-002 — отменена скалодромом
    Дано бронь имеет статус «отменена скалодромом»
    Тогда видна причина отмены
    И кнопка «Отменить запись» отсутствует

  Сценарий: AC-003 — завершённая бронь без оценки
    Дано тренировка завершена и не оценена
    Тогда видна кнопка «Оценить инструктора», ведущая на BS-004

  Сценарий: AC-004 — завершённая бронь с оценкой
    Дано клиент уже оценил инструктора
    Тогда видна поставленная оценка в режиме «только чтение»
```
