# Требования на дизайн · BS-003 · Подтверждение отмены

**ID:** BS-003 · **Тип:** Модалка · **Зона:** АЗ · **Приоритет:** High · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Foundations](00-foundations.md) · [ФТ](../functional-requirements.md) · [Use cases](../use-cases.md)

---

## 1. Назначение и контекст
Финальное подтверждение перед отменой брони — критичное действие, закрытие только явной кнопкой (не по клику на бэкдроп), чтобы избежать случайной отмены.

## 2. Навигация
- Входящая: «Отменить запись» на [SCR-006](SCR-006-booking-details.md).
- Исходящая: подтверждение/отказ → закрытие → SCR-006 (с обновлённым статусом).

## 3. Контент-инвентарь
| # | Элемент | Назначение |
|---|---|---|
| 1 | Заголовок «Отменить запись?» | — |
| 2 | Напоминание о правиле 2 часов (текст из foundations §6) | Информированное решение. |
| 3 | Индикатор, ранняя или поздняя будет отмена (по текущему времени до старта) | Прозрачность последствия. |
| 4 | Кнопка «Да, отменить» (основная) | Подтверждение. |
| 5 | Кнопка «Не отменять» (вторичная) | Отказ, закрыть без действия. |

## 4. Структура
```
┌─────────────────────────────┐
│      Отменить запись?         │
│  Тренировка через 4 часа.     │
│  Отмена сейчас — ранняя,      │
│  место освободится.           │
│  ┌───────────────────────┐   │
│  │    Да, отменить        │   │
│  └───────────────────────┘   │
│  ┌───────────────────────┐   │
│  │     Не отменять        │   │
│  └───────────────────────┘   │
└─────────────────────────────┘
```
Если до старта < 2 часов — текст меняется: «Отмена сейчас — поздняя, место не освободится (штрафов нет)».

## 5. Поведение
- Закрытие **только** по кнопке (не по клику на бэкдроп/Esc) — критичное подтверждение (foundations §4.3).
- Финальный статус (ранняя/поздняя) определяет **сервер**; если клиентский прогноз разошёлся с ответом сервера — экран-родитель показывает то, что вернул сервер.

## 6. Критерии приёмки
```gherkin
Функция: Подтверждение отмены (BS-003)

  Сценарий: AC-001 — ранняя отмена
    Дано до старта тренировки ≥ 2 часов
    Когда клиент подтверждает отмену
    Тогда бронь отменяется, место освобождается
    И SCR-006 показывает сообщение «Бронь отменена»

  Сценарий: AC-002 — поздняя отмена
    Дано до старта тренировки < 2 часов
    Когда клиент подтверждает отмену
    Тогда бронь помечается «поздняя отмена», место не освобождается
    И SCR-006 показывает соответствующее сообщение без указания на штраф

  Сценарий: AC-003 — отказ от отмены
    Когда клиент нажимает «Не отменять»
    Тогда модалка закрывается без изменения статуса брони
```
