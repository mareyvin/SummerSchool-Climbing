# Требования на дизайн · SCR-002 · Список тренировок

**ID:** SCR-002 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Foundations](00-foundations.md) · [ФТ](../functional-requirements.md) · [Use cases](../use-cases.md)

---

## 1. Назначение и контекст
Стартовый экран авторизованной зоны. Клиент видит доступные тренировки на ближайшие 7 дней и
выбирает подходящую. Главная точка входа в основной сценарий записи (UC-1).

## 2. Пользователь и цель
- **Цель клиента:** быстро найти тренировку на удобное время/формат.
- **Цель продукта:** дать полный, актуальный и не перегруженный обзор расписания.

## 3. Навигация
### Входящие
- Вход/регистрация ([SCR-001](SCR-001-registration.md)); нижняя навигация «Тренировки» с любого экрана АЗ.
### Исходящие
- Карточка слота → [SCR-003](SCR-003-slot-card.md).
- Кнопка «Фильтры» → [BS-001](BS-001-filters.md).
- Нижняя навигация → [SCR-005](SCR-005-my-bookings.md), [SCR-007](SCR-007-profile.md).

## 4. Контент-инвентарь

| # | Элемент | Источник данных | Назначение |
|---|---------|-----------------|------------|
| 1 | Заголовок «Тренировки» + кнопка «Фильтры» (с индикатором активных) | — | Заголовок + вход в BS-001. |
| 2 | Карточка слота (повторяющийся элемент списка) | `slot.*` | Краткая сводка одной тренировки. |
| 2.1 | Дата и время старта | `slot.start_at` | Когда. |
| 2.2 | Зона/формат + бейдж «Для новичков» (если применимо) | `slot.zone`, `slot.is_beginner` | Что за тренировка. |
| 2.3 | Инструктор + средний рейтинг | `slot.instructor.name`, `slot.instructor.avg_rating` | Кто ведёт (FR-41). |
| 2.4 | Свободно / всего мест | `slot.free_seats`/`slot.capacity` | Доступность. |
| 2.5 | Статус-бейдж («Мест нет» / «Отменена скалодромом») | `slot.status` | Прозрачность состояния (FR-9). |
| 3 | Empty state | — | «Пока нет доступных тренировок». |

## 5. Структура и иерархия

```
┌─────────────────────────────┐
│ Тренировки          [Фильтры•]│  ← • = индикатор активных фильтров
├─────────────────────────────┤
│ ┌─────────────────────────┐ │
│ │ Пн, 7 июля · 18:00       │ │
│ │ Болдеринг · Для новичков │ │
│ │ Инструктор: Анна ★4.8    │ │
│ │ Свободно: 3 из 8         │ │
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │
│ │ Пн, 7 июля · 20:00       │ │
│ │ Трассы с верёвкой        │ │
│ │ Инструктор: Игорь ★4.5   │ │
│ │ Мест нет                 │ │  ← бейдж, карточка тускнее, но видна
│ └─────────────────────────┘ │
│           ...                │
└─────────────────────────────┘
│ [Тренировки] [Мои записи] [Профиль] │  ← нижняя навигация
```
На `lg:` — карточки в сетке 2–3 колонки вместо одной ленты.

## 6. Компоненты и поведение
- Список отсортирован по времени старта (по возрастанию); дефолтное окно — 7 дней вперёд от `now`.
- Заполненные и отменённые скалодромом слоты **остаются в списке**, с неактивной карточкой/CTA.
- Тап по карточке (кроме заблокированных состояний) → [SCR-003](SCR-003-slot-card.md).
- Активные фильтры показываются бейджем/счётчиком на кнопке «Фильтры».
- Обновление списка — pull-to-refresh на мобильном / кнопка «Обновить» на десктопе.

## 7. Состояния экрана

| Состояние | Что показываем |
|---|---|
| Loading | Скелетоны карточек (3–5 шт.) |
| Content | Список карточек |
| Empty | «Пока нет доступных тренировок» + подсказка изменить фильтры (если применены) |
| Error | Заглушка + «Обновить» |

## 8. Валидации и микрокопия
| Правило | Текст |
|---|---|
| Пустой список без фильтров | «Пока нет доступных тренировок» |
| Пустой список с фильтрами | «Ничего не найдено. Попробуйте изменить фильтры» + кнопка «Сбросить фильтры» |
| Оффлайн-кэш | «Данные могут быть неактуальны» (см. foundations §8.3) |

## 9. Доступность
- Бейджи статуса дублируются текстом, не только цветом.
- Карточки — фокусируемые элементы с доступным именем («Тренировка, понедельник 18:00, болдеринг для новичков, свободно 3 из 8»).

## 10. Критерии приёмки дизайна
```gherkin
Функция: Список тренировок (SCR-002)

  Сценарий: AC-001 — дефолтный список на 7 дней
    Дано клиент открывает SCR-002 без заданных фильтров
    Тогда видны слоты на ближайшие 7 дней, отсортированные по времени старта

  Сценарий: AC-002 — заполненный слот виден, но неактивен
    Дано у слота free_seats = 0
    Тогда карточка показывает бейдж «Мест нет»
    И тап по карточке не открывает запись (либо открывает SCR-003 в режиме просмотра без CTA)

  Сценарий: AC-003 — отменённый скалодромом слот
    Дано слот отменён скалодромом
    Тогда карточка показывает бейдж «Отменена скалодромом»
    И повторная запись недоступна

  Сценарий: AC-004 — пустое расписание
    Дано на ближайшие 7 дней нет тренировок
    Тогда показан empty state «Пока нет доступных тренировок»
```
