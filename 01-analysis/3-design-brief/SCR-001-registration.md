# Требования на дизайн · SCR-001 · Регистрация / Вход

**ID:** SCR-001 · **Тип:** Экран · **Зона:** НЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Foundations](00-foundations.md) · [ФТ](../functional-requirements.md) · [User stories](../user-stories.md)

> Сквозные правила (тач-зоны, контраст, паттерн состояний, tone of voice) — в

## 1. Назначение и контекст
Единственная точка входа в приложение для неавторизованного пользователя. Должна снять
барьер входа: без пароля, минимум полей (P3). Клиент может быть в зале, спешит начать запись.

## 2. Пользователь и цель
- **Пользователь:** новый или уже зарегистрированный клиент.
- **Цель:** попасть в приложение максимально быстро, чтобы перейти к записи.

## 3. Навигация
### Входящие
- Первый заход по ссылке / истёкшая или отсутствующая сессия → редирект сюда с любого экрана АЗ.
### Исходящие
- Успешный вход/регистрация → [SCR-002](SCR-002-slot-list.md) (список тренировок).

## 4. Контент-инвентарь

| # | Элемент | Назначение |
|---|---------|------------|
| 1 | Логотип/название скалодрома | Контекст, доверие. |
| 2 | Переключатель способа входа: «Телефон» / «Telegram» | Выбор канала (FR-1, FR-2). |
| 3 | Поле «Номер телефона» + кнопка «Получить код» | Шаг 1 входа по телефону. |
| 4 | Поле «Код из SMS» (4–6 знаков) + таймер повторной отправки | Шаг 2, подтверждение (NFR-19). |
| 5 | Поле «Имя» (только для нового клиента) | Минимально необходимые данные о клиенте. |
| 6 | Кнопка «Войти через Telegram» | Альтернативный способ входа (FR-2), открывает Telegram-виджет/deep link. |
| 7 | Ссылка на политику конфиденциальности + чекбокс согласия | NFR-20 (ПДн), обязателен перед отправкой номера. |

## 6. Компоненты и поведение
- **Валидация номера** — маска ввода (российский формат), CTA «Получить код» активен только
  при валидном номере и отмеченном согласии.
- **Код из SMS** — автофокус на поле сразу после перехода; при вводе последнего знака —
  автоотправка (без отдельной кнопки «Подтвердить»).
- **Повторная отправка кода** — ограничена по NFR-19 (не чаще раз в 30–60 с); таймер обратного
  отсчёта на кнопке.
- **Telegram-вход** — по нажатию открывает стандартный Telegram Login Widget/OAuth-подобный
  поток; при успехе — сразу переход на SCR-002 (без шага «Имя», если Telegram уже передал имя).
- **Разрешение на Web Push НЕ запрашивается здесь** (см. 00-foundations §8.1) — только после
  первой записи на [BS-002](BS-002-booking-success.md).

## 7. Состояния экрана

| Состояние | Применимость | Примечание |
|-----------|--------------|------------|
| Content | Да | Форма ввода — основной режим. |
| Loading | Да | Во время отправки номера / проверки кода — блокировка полей + спиннер на кнопке. |
| Empty | Нет | Не применимо. |
| Error | Да | Неверный код (`invalid_code`), лимит запросов (`429`), сетевая ошибка — см. §8. |

## 8. Валидации и микрокопия

| Ошибка / код | Сообщение | UX-реакция |
|---|---|---|
| Невалидный номер | «Проверьте номер телефона» | Подсветка поля, CTA неактивен |
| `invalid_code` | «Неверный код. Попробуйте ещё раз» | Поле кода очищается, фокус остаётся |
| `429` (лимит попыток/запросов) | «Слишком много попыток. Повторите через N сек.» | Кнопка повторной отправки неактивна до истечения `retry_after` |
| Сетевая ошибка | из foundations §6 | Кнопка повтора |

## 9. Доступность
- Поля с явными `label`, маска номера объявляется screen reader'у понятным образом.
- Таб-порядок: номер → согласие → «Получить код» → «Войти через Telegram».
- Все действия доступны с клавиатуры (Enter отправляет форму).

## 10. Критерии приёмки дизайна

```gherkin
Функция: Вход и регистрация клиента (SCR-001)

  Сценарий: AC-001 — вход по телефону, новый клиент
    Дано клиент ввёл валидный номер и получил код
    Когда вводит верный код
    И это первый вход (клиент не найден по номеру)
    Тогда показывается шаг «Имя»
    И после ввода имени клиент попадает на SCR-002

  Сценарий: AC-002 — вход по телефону, существующий клиент
    Дано клиент ввёл валидный номер и верный код
    И клиент уже зарегистрирован
    Тогда шаг «Имя» пропускается
    И клиент сразу попадает на SCR-002

  Сценарий: AC-003 — вход через Telegram
    Когда клиент нажимает «Войти через Telegram» и подтверждает в Telegram
    Тогда клиент попадает на SCR-002

  Сценарий: AC-004 — неверный код
    Дано клиент ввёл неверный код
    Тогда показывается сообщение «Неверный код. Попробуйте ещё раз»
    И поле кода доступно для повторного ввода

  Сценарий: AC-005 — согласие обязательно
    Дано согласие с политикой не отмечено
    Тогда кнопка «Получить код» неактивна
```

## 11. Решения
| # | Решение |
|---|---------|
| 1 | Пароль не используется в принципе (FR-1, FR-2). |
| 2 | Запрос разрешения на Web Push — не здесь, а на BS-002 после первой брони. |
| 3 | Маскирование номера на экране ввода кода — обязательно (NFR-20). |
