## 1. Назначение и контекст
Финальный шаг перед созданием брони: выбор снаряжения и подтверждение. Здесь же обрабатываются все ошибки записи (UC-1, E1–E6) — до [BS-002](BS-002-booking-success.md) ошибочный сценарий не доходит.

## 2. Навигация
### Входящие
- «Записаться» на [SCR-003](SCR-003-slot-card.md).
### Исходящие
- Успешное подтверждение → [BS-002](BS-002-booking-success.md).
- «Назад» → SCR-003.

## 5. Компоненты и поведение
- Радио-выбор «Своё/Прокат», по умолчанию — «Своё» (не выбрано автоматически «Прокат»).
- Плашка о нехватке инвентаря — **постоянно видима**, если `rental_available` близко к исчерпанию, независимо от выбора (FR-20), не только при выборе «Прокат».
- Клиент обязан передать `Idempotency-Key` при отправке (техническая деталь реализации, не элемент UI, но важна для повторных нажатий — кнопка блокируется после первого клика на время запроса).
- При ошибке — сообщение по матрице ошибок (см. use-cases.md → UC-1), форма остаётся заполненной, повтор доступен.

## 6. Состояния
| Состояние | Примечание |
|---|---|
| Content | Форма выбора снаряжения. |
| Loading | Кнопка «Подтвердить» — спиннер, форма заблокирована на время запроса. |
| Error | Сообщение об ошибке по коду ответа (см. §7), форма остаётся. |

## 7. Валидации и микрокопия (матрица ошибок)
| Код | Сообщение | UX-реакция |
|---|---|---|
| `slot_full` | «Мест не осталось» | Обновить карточку, предложить вернуться к списку |
| `rental_unavailable` | «Прокатного снаряжения не осталось» | Предложить выбрать «Своё» |
| `beginner_flag_required` | «Тренировка только для новичков» | Заблокировать подтверждение |
| `double_booking` | «Вы уже записаны на эту тренировку» | Ссылка на «Мои бронирования» |
| `slot_cancelled` / `slot_started` | «Тренировка недоступна для записи» | Вернуться к списку |
| Сетевая/серверная ошибка | из foundations §6 | Повтор с тем же ключом идемпотентности |

## 8. Критерии приёмки
```gherkin
Функция: Оформление записи (SCR-004)

  Сценарий: AC-001 — успешная запись со своим снаряжением
    Дано клиент выбрал «Своё снаряжение» и нажал «Подтвердить запись»
    Тогда бронь создаётся и клиент переходит на BS-002

  Сценарий: AC-002 — успешная запись с прокатом
    Дано клиент выбрал «Прокат» и нажал «Подтвердить запись»
    Тогда бронь создаётся с учётом проката, прокатный фонд слота уменьшается
    И клиент переходит на BS-002

  Сценарий: AC-003 — гонка запросов, места закончились
    Дано пока клиент оформлял запись, места закончились
    Когда клиент нажимает «Подтвердить запись»
    Тогда показывается ошибка «Мест не осталось», бронь не создаётся

  Сценарий: AC-004 — плашка о нехватке проката видна всегда
    Дано прокатного фонда может не хватить на всех
    Тогда плашка-предупреждение видна независимо от выбора снаряжения
```
