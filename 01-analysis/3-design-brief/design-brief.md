## Цель и контекст

**«Вертикаль»** заменяет ручную запись на тренировки через Telegram и бумажную тетрадь,
устраняя двойные брони и путаницу с местами. Клиент самостоятельно просматривает тренировки,
фильтрует, записывается, выбирает своё или прокатное снаряжение, отменяет записи, получает
напоминания и оценивает инструктора после посещения. Платформа — **адаптивное веб-приложение
(React + Tailwind CSS)**, mobile-first, без установки из магазина приложений; открывается по
ссылке в браузере на телефоне, планшете или десктопе.

**Скоуп — только роль «Клиент».** Инструктор и владелица работают через существующую
инфраструктуру/админку и в приложение не входят. Справочные данные — read-only из API;
оплата — офлайн (наличные / перевод), приложение лишь показывает тариф проката и фиксирует
запись.

**Что входит в MVP (принятые решения):**
- **Профиль клиента (FR-33, FR-34):** просмотр и редактирование имени и телефона, привязка
  Telegram, выход из аккаунта — экран [SCR-007](SCR-007-profile.md).
- **Снаряжение (FR-18–FR-21):** выбор «своё / прокат» на уровне каждой брони; прокат имеет
  отдельный тариф, оплата — офлайн. Доступность мест и проката считается **раздельно**
  (00-foundations §6).
- **Ограничение по флагу «новичок» (FR-16):** запись на новичковые тренировки доступна только
  клиентам с этим флагом.
- **Отмена (FR-27, FR-28):** только **целиком**; порог 2 часа определяет раннюю/позднюю отмену.
- **Оценка инструктора (FR-40, FR-41):** разовая оценка 1–5 звёзд + комментарий после
  посещённой тренировки; средний рейтинг инструктора виден в карточке тренировки.
- **Уведомления (FR-45, FR-46, FR-48):** Web Push + Telegram, Telegram — основной канал.

Подробнее о продукте, аудитории, дизайн-принципах и контексте использования —
[00-foundations.md §1–2](00-foundations.md#1-продукт-и-аудитория).

## Роли и их задачи

| Роль | Что делает в приложении |
| :-- | :-- |
| **Клиент** | Просматривает и фильтрует тренировки, записывается, выбирает снаряжение, отменяет записи, видит свои брони, оценивает инструктора, получает напоминания. |
| Инструктор / Владелица | **Не входят в приложение** — работают через существующую инфраструктуру/админку (вне скоупа). |

## Сквозные правила (foundations)

Единые для всех экранов требования вынесены в **[00-foundations.md](00-foundations.md)**:
дизайн-принципы, структурные токены (контраст, тач-зоны, брейкпоинты — без бренда), каркас и
навигация, сквозной паттерн состояний (Loading / Content / Empty / Error), tone of voice и
общая микрокопия, доступность, уведомления, безопасность данных, глоссарий.

> **Объём визуальных требований.** Документы задают **функционально-структурные** требования
> (иерархия, компоненты, поведение, ограничения), без прескриптивных цветов/шрифтов — бренд
> в исходной аналитике не зафиксирован; визуальный язык определяет дизайнер.

## Пользовательские сценарии (user flows)

- **Вход (трёхшаговый):** запуск → SCR-001 (телефон → код из SMS или Telegram → имя для нового) → SCR-002.
- **Запись (UC-1):** SCR-002 → SCR-003 → SCR-004 → BS-002 → (SCR-005). ≤ 3 экранов до подтверждения (NFR-2). После **первой** записи в BS-002 запрашивается разрешение на Web Push.
- **Фильтрация (UC-3):** SCR-002 → BS-001 → SCR-002 (с применёнными фильтрами).
- **Отмена (UC-2):** SCR-005 → SCR-006 → BS-003 → SCR-006 (с обновлённым статусом). Отмена — только целиком.
- **Оценка инструктора (UC-4):** SCR-005/SCR-006 (завершённая тренировка) → BS-004 → SCR-006 (оценка отправлена).
- **Профиль:** SCR-007 — просмотр/редактирование имени и телефона, привязка Telegram, выход; выход → SCR-001.

## Состояния экранов

Единый паттерн для всех экранов с запросами — Loading (скелетон/шиммер) → Content →
Empty (заглушка + подсказка) → Error (+ «Обновить»). Описан один раз в
[00-foundations.md §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана); специфичные
состояния (disabled CTA «Записаться» при 0 мест, бейдж «Поздняя отмена» и т.п.) — в
соответствующих экранных документах.

## Тон, бренд, ограничения

- **Тон:** простой, прямой, дружелюбный, без давления и штрафов; обращение на «вы» —
  [00-foundations.md §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).
- **Бренд:** не зафиксирован в аналитике — визуальный язык за дизайнером; обязательны лишь
  структурные ограничения (контраст, размеры, иерархия, брейкпоинты) —
  [§3](00-foundations.md#3-структурные-токены-без-бренда).
- **Доступность / mobile-first веб:** крупные тач-зоны на мобильном, высокий контраст,
  клавиатурная навигация — [§3, §7](00-foundations.md#7-доступность-nfr-25-wcag-21-aa).
- **Что НЕ делаем на старте (Phase 2+):** онлайн-оплата, лист ожидания, программа лояльности,
  публичные отзывы (кроме среднего рейтинга инструктора), карта/геолокация (одна площадка),
  интерфейсы инструктора/владелицы — см. [functional-requirements.md → «Вне скоупа»](../functional-requirements.md).
