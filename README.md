# Вертикаль — запуск проекта

Моно-репозиторий: фронтенд и бэкенд лежат рядом в одном git-репозитории.

## Структура проекта

```
vertikal/
├── README.md                     ← этот файл
├── frontend/
│   └── vertikal-demo.html        ← демо-клиент (React+Tailwind по CDN, без сборки)
├── backend/
│   └── ...                       ← существующий код бэкенда (см. «Как понять, на чём это написано»)
└── 01-analysis/                  ← аналитика, требования, API-спека, ТЗ на экраны
```

> Если `frontend/` или `backend/` ещё не существуют как папки — просто создайте их и
> перенесите туда файлы. Главное — не смешивать код фронта и бэка в одной куче.

## 1. Запуск фронтенда

Файл `frontend/vertikal-demo.html` — самодостаточный демо-клиент. Сборка (npm/Vite) не нужна:
React, Tailwind и шрифты подключены по CDN, JSX компилируется прямо в браузере.

### Вариант A — просто открыть файл (самый быстрый)

Дважды кликните `frontend/vertikal-demo.html` — откроется в браузере через `file://`.

Ограничение: если в файле прописан реальный адрес бэкенда (`CONFIG.API_BASE`), часть браузеров
блокирует `fetch`-запросы со страниц, открытых через `file://` (CORS). Для показа на моках —
не важно, для реального запроса к бэкенду — используйте вариант B.

### Вариант B — через локальный сервер (надёжнее, меньше проблем с CORS)

```bash
cd frontend
python3 -m http.server 8000
# либо, если нет python:
npx serve .
```

Откройте `http://localhost:8000/vertikal-demo.html`.

### Подключение к бэкенду

В начале `vertikal-demo.html` есть блок:

```js
window.CONFIG = {
  API_BASE: "http://localhost:3000",   // ← адрес вашего бэкенда
  SLOTS_PATH: "/slots",                // ← путь к списку тренировок
  FETCH_TIMEOUT_MS: 3500
};
```

Если бэкенд недоступен, ответит с ошибкой или зависнет дольше 3.5 секунд — фронт **сам**
переключится на встроенные моковые данные (внизу списка тренировок это видно по индикатору
🟢/🟡). Так демо не сломается прямо во время показа, даже если бэк упадёт.

## 2. Запуск бэкенда

### Если вы не уверены, на чём он написан — определяем за 30 секунд

Зайдите в папку `backend/` и посмотрите, какой файл там есть:

| Что нашли в папке | Скорее всего стек | Команда запуска |
| :-- | :-- | :-- |
| `package.json` | Node.js (Express/Fastify/NestJS) | `npm install && npm run dev` (или `npm start`) |
| `requirements.txt` или `pyproject.toml` | Python (FastAPI/Flask/Django) | см. ниже, раздел Python |
| `manage.py` | Python/Django | `python manage.py runserver` |
| `go.mod` | Go | `go run .` |
| `pom.xml` или `build.gradle` | Java/Kotlin | `./mvnw spring-boot:run` или `./gradlew bootRun` |
| `Gemfile` | Ruby | `bundle exec rails server` |
| `composer.json` | PHP | `php artisan serve` (если Laravel) |
| `docker-compose.yml` | Любой — но уже расписан для вас | `docker compose up` |

Быстрая команда, чтобы увидеть все "маркерные" файлы сразу:

```bash
cd backend
ls -a
```

### Node.js — конкретные шаги

```bash
cd backend
npm install
cp .env.example .env   # если есть — заполните переменные (порт, строка подключения к БД и т.д.)
npm run dev            # или: npm start / node index.js / node server.js — смотрите package.json → "scripts"
```

Проверьте, на каком порту поднимается сервер (обычно видно в консоли при старте, часто `3000`
или `4000`) — этот порт указывайте во фронтенде в `CONFIG.API_BASE`.

### Python — конкретные шаги

```bash
cd backend
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
# FastAPI:
uvicorn main:app --reload --port 8000
# Flask:
flask run --port 8000
# Django:
python manage.py migrate
python manage.py runserver 8000
```

### Если есть база данных (Postgres/MySQL) и она тоже не поднята

```bash
# Проще всего — если есть docker-compose.yml с описанием БД:
docker compose up -d db
# Тогда бэкенду нужна переменная окружения с адресом БД, обычно в .env:
# DATABASE_URL=postgresql://user:password@localhost:5432/dbname
```

Если docker-compose нет и БД настраивать некогда — для демо честнее временно фронт полностью
перевести на моки (`CONFIG.API_BASE = ""`, тогда `fetch` сразу упадёт и подставятся моковые
данные) и не пытаться чинить БД за оставшееся время.

## 3. Проверка перед показом (чек-лист)

- [ ] Открывается `frontend/vertikal-demo.html` (или через `http://localhost:8000`)
- [ ] Видно индикатор источника данных (🟢 реальные / 🟡 демо) — знаете заранее, что покажете
- [ ] Флоу «Тренировки → карточка слота → запись → мои бронирования → отмена» проходит без ошибок в консоли (F12 → Console)
- [ ] Если бэкенд нестабилен — заранее решите: показываете попытку реального запроса (с фолбэком на моки, это нормально и честно) или сразу отключаете `API_BASE`, чтобы не рисковать зависанием на 3.5 сек посреди демо
